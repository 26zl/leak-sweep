# Security policy

The current `1.x` release line receives security fixes.

Report vulnerabilities privately through this repository's GitHub Security
Advisories. Do not put real secrets, credentials, national IDs or other personal
data in an issue, pull request, screenshot or test fixture.

If the scanner identifies an active credential, revoke or rotate it first. Then
remove it from the current tree and, when exposure through history matters,
rewrite the affected Git history before making the repository public again.
